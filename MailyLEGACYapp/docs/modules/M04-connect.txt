--------------------------------------------------------------------------------
PAYLOAD DE REGISTRO (POST /api/v1/vitals/)
--------------------------------------------------------------------------------
Campo           | Tipo     | Req | Notas
----------------|----------|-----|--------------------------------------------
vital_type      | string   |  ✓  | Enum: BLOOD_PRESSURE|HEART_RATE|OXYGEN_SAT|
                |          |     | RESPIRATORY|GLUCOSE|GLUCOSE_FAST|WEIGHT|
                |          |     | HEIGHT|WAIST|HIP|TEMPERATURE|STEPS|SLEEP_HOURS
value           | number   |  ✓  | Valor primario (sistólica si BLOOD_PRESSURE)
secondary_value | number   |  *  | Solo BLOOD_PRESSURE → diastólica (requerido)
recorded_at     | ISO8601  |  ✓  | Fecha/hora del registro
source          | string   |     | MANUAL (default) | DEVICE | INTEGRATION
notes           | string   |     | Texto libre (max 300 chars)
photo_url       | string   |     | URL pública en R2 (flujo presigned upload)

* secondary_value: requerido ÚNICAMENTE para BLOOD_PRESSURE.
  Si se envía en otro tipo → 400 "secondary_value solo aplica para BLOOD_PRESSURE."

--------------------------------------------------------------------------------
FLUJO FOTO DE EVIDENCIA
--------------------------------------------------------------------------------
1. Usuario toca "📷 Adjuntar foto" en la tarjeta del signo
2. expo-image-picker → galería (permiso media library requerido)
3. POST /api/v1/documents/upload-url/ con { file_name, mime_type }
   → responde { upload_url (presigned PUT), file_url (URL pública R2) }
4. PUT directo a upload_url con el blob de la imagen
5. file_url se guarda en photoUrls[type] (estado local)
6. Al hacer POST /api/v1/vitals/ → photo_url: file_url incluido en payload
