HOOKS
--------------------------------------------------------------------------------
src/hooks/useProfile.ts
  usePatientProfile()    → perfil del paciente autenticado
  useDoctorProfile()     → perfil del doctor autenticado
  useUpdateProfile()     → mutación PATCH → invalida ['profile']
  useUploadPhoto()       → POST foto → URL en R2 guardada en DB
  useDoctorPatients()    → lista de pacientes del doctor (usePatients.ts)

--------------------------------------------------------------------------------
FLUJO DE FOTO DE PERFIL
--------------------------------------------------------------------------------
1. Usuario selecciona imagen (expo-image-picker)
2. POST /auth/profiles/patient/photo/ con FormData (multipart)
3. Backend sube a R2, devuelve { photo_url }
4. El store actualiza user.photoUrl via updateUser()

--------------------------------------------------------------------------------
NOTAS TÉCNICAS
--------------------------------------------------------------------------------
- El perfil del paciente incluye: fecha de nacimiento, tipo de sangre, alergias,
  condiciones crónicas, contacto de emergencia.
- Estos campos son sensibles: no se loguean en Sentry (sanitizados en before_send).
- El doctor asigna pacientes por email desde el panel; el frontend (doctor) solo
  los lista y consulta, no los asigna.
- SpecialistProfile y PartnerProfile no tienen pantalla de edición completa en el
  móvil en esta versión — se gestionan desde el admin de Django.
