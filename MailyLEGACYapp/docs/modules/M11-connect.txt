ENDPOINTS CONECTADOS
--------------------------------------------------------------------------------
EP.documentUploadUrl = POST /api/v1/documents/upload-url/
  Parámetros: { file_name, mime_type }
  Devuelve:   { upload_url, file_url, key }
  Uso: subir directamente a R2, luego pasar file_url al recurso

EP.documents = GET /api/v1/documents/
  Parámetros opcionales: ?category=LAB_RESULT|PRESCRIPTION|IMAGING|CLINICAL_NOTE|INSURANCE|OTHER
  Devuelve lista de MedicalDocument del paciente autenticado
  Usado en: documents/index.tsx (con filtro por categoría)

EP.documents = POST /api/v1/documents/
  Body: { title, category, file_url, file_name, file_size?, mime_type? }
  Registra el documento médico (dispara OCR async)
  Usado en: documents/index.tsx (confirmUpload)

DELETE /api/v1/documents/<id>/
  Soft delete (is_active = False)
  Usado en: documents/index.tsx (confirmDelete → swipe or trash icon)

--------------------------------------------------------------------------------
FLUJO DE UPLOAD (presigned R2)
--------------------------------------------------------------------------------
1. Usuario elige archivo con expo-document-picker (PDF o imagen)
2. Modal solicita título y categoría
3. POST /documents/upload-url/ → { upload_url, file_url }
4. PUT directo a R2 con blob del archivo (Content-Type correcto)
5. POST /documents/ con { title, category, file_url, file_name, file_size, mime_type }
6. Backend crea MedicalDocument + encola process_document_ocr()
7. Lista se invalida con queryClient.invalidateQueries(['documents'])
