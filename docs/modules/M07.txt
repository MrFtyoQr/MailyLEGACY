--------------------------------------------------------------------------------
DECISIONES DE DISEГ‘O
--------------------------------------------------------------------------------
1. notify() es la Гєnica API que otros mГіdulos deben usar вЂ” encola dispatch_notification.
2. Canal seleccionado segГєn preferencias del usuario (futuro) y tipo de notificaciГіn.
3. Los tokens de dispositivo son multi-dispositivo por usuario (tablet + celular).
4. PUSH usa firebase-admin (FCM v1 API) para Android y APNs via FCM para iOS.
5. EMAIL usa SendGrid como proveedor principal; SES como fallback (configurable via env).
6. Las notificaciones IN_APP se guardan siempre en la base de datos; PUSH y EMAIL
   solo si el usuario tiene token o email verificado.
7. Soft retry en dispatch_notification: hasta 3 reintentos con backoff exponencial.
