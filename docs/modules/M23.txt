--------------------------------------------------------------------------------
DESCRIPCIÓN
--------------------------------------------------------------------------------
Módulo de selección y gestión de planes de suscripción dentro de la app móvil
(React Native / Expo). Permite al paciente comparar y contratar los planes
FREE, SILVER, GOLD y PLATINUM en modalidad Individual o Familiar.

El módulo conecta con el backend de pagos (Stripe via Django) a través de los
endpoints ya existentes en /payments/.

--------------------------------------------------------------------------------

STACK DEL MÓDULO (App)
--------------------------------------------------------------------------------
- React Native / Expo (expo-router v6)
- NativeWind / StyleSheet
- @tanstack/react-query v5 (usePlans, useSubscription, useCheckout)
- expo-linear-gradient (gradientes por tier)
- Zustand — auth.store (tier del usuario)
- Linking (abrir checkout Stripe en browser externo)

--------------------------------------------------------------------------------
ENDPOINTS UTILIZADOS
--------------------------------------------------------------------------------
GET   /payments/plans/          → lista de planes disponibles (backend)
GET   /payments/subscription/   → suscripción activa del usuario
POST  /payments/checkout/       → genera URL de checkout Stripe
POST  /payments/cancel/         → cancela suscripción activa

--------------------------------------------------------------------------------
DECISIONES DE DISEÑO
--------------------------------------------------------------------------------
1. Precios locales (FEATURE_MATRIX) como fallback — la UI funciona sin backend.
2. El checkout abre una URL de Stripe en el browser del dispositivo via Linking.
3. Cards con acordeón: solo un tier abierto a la vez (default: GOLD).
4. El tier actual se resalta con borde verde y badge "Activo".
5. Toggle Individual/Familiar cambia los precios en tiempo real.
6. No se implementó cancelación en esta pantalla — queda en la siguiente iteración.
