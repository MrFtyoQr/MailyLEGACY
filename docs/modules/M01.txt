--------------------------------------------------------------------------------
DESCRIPCIÓN
--------------------------------------------------------------------------------
Módulo de autenticación y seguridad. Maneja identidad de usuarios via Clerk,
sincronización local en PostgreSQL, control de roles, protección brute force,
recuperación de contraseña y auditoría de accesos.

--------------------------------------------------------------------------------
STACK DEL MÓDULO
--------------------------------------------------------------------------------
- Clerk (JWT, MFA, Google Sign-In, recuperación contraseña)
- Django REST Framework (endpoints de sincronización y perfil)
- clerk-backend-api (Python SDK para verificación JWKS)
- django-ratelimit (segunda capa anti brute force)
- Sentry SDK (logging de errores de auth)

--------------------------------------------------------------------------------
DECISIONES DE DISEÑO
--------------------------------------------------------------------------------
1. Clerk maneja: registro, login, MFA, Google Sign-In, recuperación de contraseña.
2. Django solo almacena clerk_id como FK — datos médicos NUNCA tocan Clerk.
3. Roles asignados en publicMetadata de Clerk al crear usuario vía webhook.
4. Rate limiting: 5 intentos fallidos por IP en 15 min (django-ratelimit).
5. JWT: 15 min access token / 7 días refresh (configurado en Clerk dashboard).
6. Roles soportados: PATIENT | DOCTOR | SPECIALIST | PARTNER | ADMIN.
7. Cada login/logout queda registrado en audit_logs_event.

--------------------------------------------------------------------------------
ENDPOINTS EXPUESTOS
--------------------------------------------------------------------------------
POST  /api/v1/auth/webhook/clerk/       → recibe eventos de Clerk (svix signature)
GET   /api/v1/auth/me/                  → perfil del usuario autenticado
PATCH /api/v1/auth/me/                  → actualizar datos básicos de perfil
GET   /api/v1/auth/me/role/             → rol actual + permisos
